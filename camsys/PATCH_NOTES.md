# CAMSYS v1.5.14 — патч (нарастающий, поверх v1.4.3)

## Что нового с v1.5.13

**D3-режим: клик по подписи заказа переключает видимость его путей.**

Смысл ✓ на подписи заказа изменён:
- Раньше: «папка `<order>-NC/` существует на диске»
- **Теперь: «пути этого заказа посчитаны и лежат в сессионном кэше
  `_order_toolpath_items`»** — независимо от того, сохранены ли файлы.

Это правильный индикатор перед общим экспортом «Сохранить всё» —
оператор видит для каких заказов уже есть готовые пути.

### Клик по подписи (D3)

Клик по подписи заказа на сцене (или по её тёмному фону) переключает
видимость путей ЭТОГО заказа НА ФОНЕ активного:

- Активный заказ (из dropdown'а) всегда видим, клик по его подписи —
  no-op (нельзя «спрятать активного»).
- Не-активный заказ: клик → пути дорисовываются к сцене (visible=True
  на кэшированных toolpath-items). Второй клик — скрывает.
- Множество «дополнительно показать» хранится в `_extra_shown_orders`,
  сохраняется при смене активного заказа (если ты покажешь 3 заказа
  сразу — они останутся видимы при переключении между ними).

### Визуальные состояния подписи

- **Белая** — путей нет, клик покажет ничего (нужно сначала «Пересчитать»).
- **Зелёная с ✓** — пути посчитаны, скрыты, готовы к экспорту.
- **Жёлтая на тёмно-жёлтом фоне** — сейчас показана на сцене
  (активная либо в `_extra_shown_orders`).

### Автообновление подписей

`_refresh_stitch_labels()` вызывается:
- После анализа сшивки (первый показ)
- После смены активного заказа в dropdown
- После пересчёта путей (`_order_toolpath_items[key]` пополнился)
- После клика по подписи (изменилось `_extra_shown_orders`)

# CAMSYS v1.5.10 — патч (нарастающий, поверх v1.4.3)

## Что нового с v1.5.9

**Drill реперов теперь только в `_all_R.anc`.**

До этого точки прошивки реперов протекали в три чужих файла:
- `_revers_R.anc` — 2 drill-линии (реверсная черновая, где реперы уже
  насверлены прямой — прошивать ещё раз нельзя)
- `_1_M.anc` — 2 drill-линии (первая чистовая программа получала
  прошивку вместе с чистовым проходом ножей program_number=1)
- `_SV.anc` — 1 drill-линия (программа контроля сведения — там вообще
  не должно быть реперов)

Причина: три генератора (`_generate_rough_all(reverse=True)`,
finish-per-op группировка в `_generate_impl`, `_generate_sv`) итерируют
`project.operations` целиком, включая `FIDUCIAL_DRILL` операции. Только
прямой `_generate_rough_all(reverse=False)` явно добавляет свой
`drill_op` через `make_fiducial_drill_operation`, а остальным реперы
попадают «сами» через существующие per-fiducial операции.

Фикс: во всех трёх генераторах фильтрую `FIDUCIAL_DRILL` из списка
операций до эмита. `_all_R` продолжает работать как раньше — он единственный
кто добавляет drill целенаправленно.

**Проверено на 122425 в 41287** (после установки настроек по одному
заказу):

```
_all_R.anc     drill=2  ← остались (правильно)
_revers_R.anc  drill=0  ✓ было 2
_1_M.anc       drill=0  ✓ было 2
_2_M.anc       drill=0
_3_M.anc       drill=0
_SV.anc        drill=0  ✓ было 1
_corner.anc    drill=0
```

Zoom к курсору, комментарии `;40,20/;50,21/;60,21` на теле контура,
POSITION-вариант с автовызовом из UI, валидация >0, фикс `_dop.anc`,
три фикса drill'а (v1.5.4), identity-skip первого заказа, поддержка
копий одного заказа, правильный порядок обхода в POSITION для
повёрнутых заказов, чередующийся порядок сдвигов при коллизиях лидов
(v1.5.8), правильная привязка реперов к регионам «репер внизу заказа»
(v1.5.9), инверсия offset по направлению обхода полипаса (v1.5.9).

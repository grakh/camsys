# CAMSYS v1.5.6 — патч (нарастающий, поверх v1.4.3)

## Что нового с v1.5.5

**Поддержка нескольких копий одного заказа в одной сшивке.**

Когда одна и та же деталь размножена по плите (например, 121254 лежит
двумя одинаковыми копиями по 66 ножей), матчер `_match_orders_to_regions`
блокировал повторные вхождения одного и того же номера — вторая копия
оставалась без имени, в дропдауне UI показывалось «регион 2».

Правки:

1. **`io_/stitch.py`**: убрал блок `if it.text in result.values(): continue`
   и в этапе 1, и в этапе 2 матчера. Один номер теперь может привязаться
   к нескольким регионам (регионы, наоборот, уникальны в `result`).

2. **`io_/stitch.py`**: `get_region_by_order` расширен — понимает
   композитный ключ «121254#i» (0-based индекс копии). Ключ без «#» —
   обратная совместимость, возвращает первый регион с таким номером.

3. **`io_/stitch.py`**: новый helper `order_key_to_filename(key)`:
   - `"121254"` → `"121254"`
   - `"121254#0"` → `"121254"` (первая копия без суффикса)
   - `"121254#1"` → `"121254_c2"` (2-я копия)
   - `"121254#2"` → `"121254_c3"` (3-я копия)

   Символ «#» в путях/именах файлов местами обрезается — используем
   «_cN» для файловых форм.

4. **`ui/main_window.py`**: дропдаун показывает суффикс «(копия N)» для
   заказов с несколькими регионами. `itemData` = композитный ключ.
   Папки NC и `output_prefix` строятся через `order_key_to_filename`.

5. **`core/session.py`**: `export_package_position` тоже применяет
   `order_key_to_filename` для prefix'а/имени файлов.

## Как теперь выглядит

Если сшивка содержит две копии 121254 плюс два уникальных заказа,
дропдаун:

- 121254 (копия 1) (66 ножей)
- 121254 (копия 2) (66 ножей)
- 121323 (14 ножей)
- 121338 (12 ножей)

Файлы для копий:
- `121254-NC/…all_R.anc`
- `121254_c2-NC/…all_R.anc`
- `121323-NC/…all_R.anc`
- `121338-NC/…all_R.anc`

У каждой копии свои реперы, свой POSITION-трансформ, независимый
пакет .anc. Первая копия имеет старый префикс `"121254"` — совместимо
с существующими файлами и папками.

## Что предположительно НЕ ломается

- Уникальные заказы (без копий): весь код идёт по старому пути,
  композитный ключ не создаётся.
- Существующие 143 теста: все проходят.

## Что важно проверить на живой сшивке с 121254

1. Дропдаун показывает «121254 (копия 1)» и «121254 (копия 2)»
   вместо «регион 2».
2. При выборе копии 1 подсвечивается один регион на сцене, копии 2 —
   другой (разные ножи, разные реперы).
3. Экспорт пишет в две разные папки: `121254-NC/` и `121254_c2-NC/`.
4. POSITION для каждой копии считает свой трансформ (у копий разные
   LB-реперы) — файлы в подпапках `POSITION/` будут разные.

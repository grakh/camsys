# Сессия 2026-06-10 (9) — сглаживание под фрезу (без самопересечений offset)

### Задача
- На 6 ножах (37,35,76,72,45,66) эквидистанта (G42 comp в вьювере) давала
  «спирали/веера» у тугих вогнутых мест и на биарк-кластерах. Осевая в .anc
  чистая (0 самопересеч), но offset фрезы 0.575мм самопересекается. Меньшая
  фреза — не вариант, нужно сглаживание геометрии под фрезу.

### Решение  [path_offset.smooth_for_offset + shapely]
- smooth_for_offset(polypath, tool_offset, side): через shapely морфологическое
  замыкание (OUTSIDE) / размыкание (INSIDE) радиусом tool_offset. Тугие места
  скругляются ровно до проходимого радиуса, биарк-кластеры исчезают (осевая →
  чистая ломаная). Намотка и старт сохраняются. Отклонение от оригинала 0–1мкм
  (меняются только реально тугие места). Без shapely — возвращает оригинал.
- Флаг smooth_offset_for_tool (CuttingMacroParams) + extras → эмиттер и
  визуализатор применяют одинаково (по запросу, галка в UI). При smooth=True
  все дуги → ломаная (G2/G3=0), .anc крупнее (~2x), но offset чистый.
- UI: галка «Сглаживание под фрезу (без самопересечений)» в «Генерировать файлы».
- requirements: + shapely>=2.0.

   под пользовательский эталон (диам=2мм + пятка=0.8 → L=0.7):

     - диам < 4×пятка → L_mm = (диам − пятка) / 4
     - диам < 8×пятка → L_mm = tool_radius
     - иначе → L_mm = 2×tool_radius
     - L = L_mm / tool_radius, округление до 0.1
   Проверка: R=1 пятка=0.8 → L=0.7 ✓
2. Автоподбор теперь вызывается АВТОМАТИЧЕСКИ при показе путей фрезы

- Иначе → Arc. Никакого спрямления по большому радиусу.
- min_chain=3 (минимум 3 сегмента в цепочке).
- В session.load_ai обрабатывается ТОЛЬКО слой Knife.
